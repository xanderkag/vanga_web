# Ошибки и валидация

Раздел описывает коды ошибок API и флаги доменной валидации, которые
приходят в результате обработки.

## Коды ошибок API

| HTTP | Когда | Что делать |
|---|---|---|
| `400` | Файл не приложен или повреждён | Проверьте `multipart`-форму и сам файл |
| `401` | Нет или неверный Bearer-токен | Проверьте заголовок `Authorization` |
| `404` | `job_id` не найден | Сверьте идентификатор |
| `409` | Действие в неверном статусе job | Дождитесь терминального статуса |
| `415` | Неподдерживаемый тип файла | Допустимы PDF, JPG, PNG |
| `422` | Не удалось распознать/классифицировать | Документ уйдёт в `needs_review` |
| `429` | Слишком много запросов | Повтор с backoff |
| `5xx` | Внутренняя ошибка | Ретрай; при повторении — в поддержку |

## Терминальные статусы job

- **`done`** — документ распознан и извлечён, валидация без критичных флагов.
- **`needs_review`** — результат есть, но валидация подняла флаги или
  `confidence` ниже порога. Требуется ручная сверка.
- **`failed`** — обработать не удалось (например, нечитаемый или защищённый
  паролем PDF).

<Callout type="warning" title="needs_review — это не ошибка">
  Статус <code>needs_review</code> означает «данные есть, но проверьте». Поля в{" "}
  <code>extracted</code> заполнены; смотрите блок <code>validation</code>, чтобы
  понять, что именно требует внимания.
</Callout>

## Флаги доменной валидации

В ответе блок `validation` содержит результат проверок по полям:

| Флаг | Значение `ok` | Значение проблемы |
|---|---|---|
| `inn` | Контрольная сумма ИНН сходится | `invalid_checksum` — ИНН не проходит проверку |
| `kpp` | Формат КПП корректен | `invalid_format` |
| `vat` | НДС консистентен с суммами | `mismatch` — НДС не сходится с итогом |
| `plate` | Госномер распознан и валиден | `unrecognized` — не удалось разобрать номер ТС |
| `totals` | Сумма позиций = итог | `mismatch` — арифметика не сходится |

Пример с проблемой консистентности НДС:

```json
{
  "status": "needs_review",
  "validation": {
    "inn": "ok",
    "vat": "mismatch",
    "totals": "ok"
  }
}
```

<Callout type="info">
  Набор флагов зависит от типа документа: для транспортных проверяется{" "}
  <code>plate</code>, для бухгалтерских — <code>inn</code>/<code>vat</code>/
  <code>totals</code>.
</Callout>
